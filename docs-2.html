Introduction

Klar is a lightweight CMS that lets you edit HTML pages visually while preserving clean, semantic markup. No bloat. No unnecessary wrappers. Just your HTML.

Zero Bloat
No injected styles, no wrapper divs, no script tags in your output.
Clean Output
Outputs semantic HTML that humans can read and maintain.
~8KB Gzipped
Minimal JavaScript footprint. Fast load times on any device.

Why Klar?#

Most CMS tools inject their own markup into your pages — wrapper divs, data attributes, inline styles, and script tags that make the output unmaintainable. Klar takes a different approach:

  • Your HTML stays yours. Klar edits are applied server-side with surgical precision.
  • CSS selectors as identifiers. No data-klar-id attributes cluttering your markup.
  • Semantic output. The HTML you get out is the HTML you'd write by hand.
Good to know: Klar uses Cheerio server-side for all HTML transformations, ensuring your output is never polluted by client-side editing artifacts.

How it Works#

Klar renders your target page in a sandboxed iframe. When you click on any text element, it becomes editable. Changes are tracked by CSS selector and applied server-side using Cheerio before being committed back to your GitHub repository.

HTML 
<!-- Your original HTML -->
<section class="hero">
  <h1>Welcome to My Site</h1>
  <p>This is the description.</p>
</section>

<!-- After editing with Klar — still clean! -->
<section class="hero">
  <h1>Welcome to My Updated Site</h1>
  <p>A better description here.</p>
</section>

Quick Overview#

Three Edit Types

Klar supports three types of edits, each applied differently:

Type Description Example
text Updates the text content of an element Changing a heading or paragraph
attribute Modifies an HTML attribute value Changing an image src or link href
outerHTML Replaces the entire element markup Restructuring a complex component

Phase-Based Flow

The app uses a state machine with three phases:

  • Setup — Configure your GitHub repository and target page
  • Loading — Fetch and sanitize the HTML from your repo
  • Editing — Visual editor with live preview and structured panel

Installation

Get Klar up and running in your project with npm or yarn.

Requirements#

  • Node.js 18 or later
  • A GitHub repository with HTML files
  • A GitHub personal access token (for the API)

Install#

Terminal 
# Clone the repository
git clone https://github.com/your-org/klar.git
cd klar

# Install dependencies
npm install

# Start the dev server
npm run dev

Environment Setup#

Create a .env file in the project root:

.env 
GITHUB_TOKEN=ghp_your_token_here
GOOGLE_CLIENT_ID=your_client_id
SESSION_SECRET=a_random_string
Important: Never commit your .env file to version control. Add it to your .gitignore.

Quick Start

Go from zero to editing your first page in under 5 minutes.

1. Start the Dev Server#

Terminal 
npm run dev

This starts both the Vite dev server (port 5173) and Express backend (port 3001) with hot reload.

2. Configure Your Project#

Enter your GitHub repository details in the setup form:

  • Repository owner — your GitHub username or org
  • Repository name — the repo containing your HTML
  • File path — path to the HTML file (e.g., index.html)
  • Branch — defaults to main

3. Start Editing#

Click any text element in the preview to edit it. Your changes are tracked automatically and can be saved back to GitHub with a single click.

Clean HTML Output

How Klar preserves your markup integrity through every edit cycle.

The Problem with Traditional CMS#

Most visual editors inject their own markup — wrapper divs, data attributes, inline styles. After a few edits, your clean HTML becomes an unmaintainable mess.

The Klar Approach#

Klar tracks edits by CSS selector and applies them server-side using Cheerio. Your HTML goes through three stages:

  1. Fetch — raw HTML from your GitHub repo
  2. Sanitize — prepare for safe iframe rendering
  3. Apply — merge edits back into original structure
Diff 
  <section class="hero">
    <h1>
-     Welcome to My Site
+     Welcome to My Updated Site
    </h1>
  </section>

HTML Sanitization

How Klar prepares HTML for safe rendering in the editor iframe.

Sanitization Process#

Before rendering in the iframe, Klar sanitizes the HTML to prevent scripts from executing and ensures safe contentEditable behavior. The process includes:

  • Removing all <script> tags
  • Neutralizing event handler attributes (onclick, etc.)
  • Wrapping naked text nodes for contentEditable compatibility
  • Rewriting relative URLs to absolute paths
Sanitization only affects the editor preview. When saving, edits are applied to the original unsanitized HTML from your repository.

Export & Save

Save your edits back to GitHub and publish with one click.

Save to GitHub#

When you click Save, Klar applies your edits to the original HTML using Cheerio and commits the changes directly to your repository. The process:

  1. Fetches the latest file content from GitHub
  2. Applies each edit by CSS selector
  3. Commits the modified file with a descriptive message

Publish#

The Publish action goes a step further: it saves your changes to a klar-publish branch and enables GitHub Pages, giving you a live preview URL instantly.

API Overview

All API endpoints accept POST requests with JSON bodies and require cookie-based authentication.

Base URL#

URL 
http://localhost:3001/api

Endpoints#

Endpoint Purpose
/auth/google Google OAuth login
/auth/me Check session, return user + projects
/fetch-page Fetch and sanitize HTML from a URL
/fetch-from-repo Fetch file from GitHub repo
/github/save Apply edits and commit to GitHub
/github/publish Save + create branch + enable Pages
/github/media/* List, upload, delete media files
/projects Create/update project

Methods

Detailed reference for each API method with request/response examples.

fetchPage#

Fetches HTML from a URL, sanitizes it for iframe rendering, and returns the processed content.

Request 
POST /api/fetch-page
Content-Type: application/json

{
  "url": "https://example.com",
  "projectId": 1
}

save#

Applies edits and commits the changes to GitHub.

Request 
POST /api/github/save
Content-Type: application/json

{
  "projectId": 1,
  "edits": [
    {
      "selector": ".hero h1",
      "type": "text",
      "value": "New Heading"
    }
  ]
}

Events

PostMessage events for parent-iframe communication.

Message Types#

Parent and iframe communicate via postMessage using discriminated union types. Each message has a type field that determines its shape.

Direction Type Purpose
Parent → Iframe highlight Highlight an element by selector
Parent → Iframe updateContent Update element content
Iframe → Parent elementClicked User clicked an editable element
Iframe → Parent contentChanged Content was modified in the iframe

Performance

How Klar stays fast with ETag caching and efficient API usage.

ETag Caching#

Klar uses ETag-based caching for all GitHub API calls. When a file hasn't changed, GitHub returns a 304 Not Modified response, saving both bandwidth and API quota.

API Quota Management#

The file cache is manually updated after successful commits, so subsequent reads don't require an API call. This keeps Klar well within GitHub's rate limits even for active editing sessions.

Accessibility

Klar is built with accessibility in mind, ensuring the editor is usable by everyone.

Keyboard Navigation#

All editor controls are keyboard accessible. Use Tab to navigate between elements and Enter to activate buttons.

ARIA Support#

The editor uses proper ARIA attributes for screen reader compatibility, including aria-label on interactive elements and role attributes on navigation landmarks.
