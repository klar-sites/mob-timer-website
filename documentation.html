Introduction

Klar is a lightweight CMS that lets you edit HTML pages visually while preserving clean, semantic markup. No bloat. No unnecessary wrappers. Just your HTML.

Zero Bloat
No injected styles, no wrapper divs, no script tags in your output.
Clean Output
Outputs semantic HTML that humans can read and maintain.
~8KB Gzipped
Minimal JavaScript footprint. Fast load times on any device.

Why Klar?#

Most CMS tools inject excessive markup into your pages. WYSIWYG editors wrap everything in <div> and <span> tags with inline styles. Klar takes a fundamentally different approach.

Core principle: Your HTML should look the same whether it was written by hand or edited with Klar. No artifacts. No tracking attributes.

When you save content in Klar, the output is indistinguishable from hand-written HTML. We strip all editor artifacts, normalize formatting, and respect your existing code style.

Before & After#

Here's what a typical CMS outputs versus what Klar produces:

html — ❌ Typical CMS output 
<div class="editor-wrapper ce-block">
  <div class="ce-block__content">
    <p style="font-weight:700;color:#333"
       data-editor-id="block-x7f9">
      <span style="font-weight:bold">Hello world</span>
    </p>
  </div>
</div>
html — ✓ Klar output 
<p><strong>Hello world</strong></p>

Design Philosophy#

Klar is built around three principles:

  1. Semantic by default — We use <strong> not <span style="font-weight:bold">.
  2. Minimal footprint — The entire runtime is under 8KB gzipped.
  3. Developer-friendly — Clean API, documented events, extensible via plugins.

Who it's for#

Klar is ideal for developers and teams who care about code quality. If you version control your HTML, work with static site generators, or simply prefer clean markup — Klar is built for you.

Installation

Add Klar to your project in seconds. No build steps required.

CDN (Recommended)#

The fastest way to get started. Add the script tag before your closing </body> tag:

html 
<script src="https://cdn.klar.dev/v0.9/klar.min.js"></script>

NPM#

For projects using a build tool or bundler:

bash 
npm install klar-cms

Then import it in your JavaScript:

javascript 
import { Klar } from 'klar-cms';

const editor = new Klar({
  target: '#content',
  mode: 'inline'
});

Quick Start

Get your first editable page running in under 2 minutes.

Step 1: Add editable regions#

Mark sections of your HTML as editable using the data-klar attribute:

html 
<article data-klar="main-content">
  <h1>Welcome to my site</h1>
  <p>Click here to edit this text.</p>
</article>

Step 2: Initialize Klar#

Add the initialization script. Klar will automatically discover all data-klar regions:

javascript 
const klar = new Klar({
  autoDiscover: true,
  sanitize: true,
  onSave: async (regions) => {
    // regions is a Map of id → clean HTML
    await fetch('/api/save', {
      method: 'POST',
      body: JSON.stringify(regions)
    });
  }
});

Step 3: That's it#

Klar handles everything else. Your users can click to edit, format with keyboard shortcuts, and save — all while the output stays clean.

The data-klar attribute is only used during editing. It is automatically removed from saved output.

Clean HTML Output

Understand how Klar sanitizes and normalizes HTML to keep your codebase clean.

Automatic Sanitization#

When you save content through Klar, every editable region goes through a multi-pass sanitization pipeline:

  1. Strip editor artifacts — Remove contenteditable, data-klar-*, and any editor-specific attributes.
  2. Normalize formatting — Convert <span style="font-weight:bold"> to <strong>, etc.
  3. Remove empty nodes — Clean up empty <span> and <div> tags left by editing.
  4. Collapse wrappers — Unwrap unnecessary nesting levels.
  5. Format output — Consistent indentation and line breaks.

Transformation Rules#

Input (dirty) Output (clean)
<span style="font-weight:bold"> <strong>
<span style="font-style:italic"> <em>
<div class="editor-block"><p>...</p></div> <p>...</p>
<p data-block-id="x7f9"> <p>
<span>text</span> (no attributes) text

Custom Rules#

You can define your own sanitization rules via the config:

javascript 
new Klar({
  sanitize: {
    allowedTags: ['p', 'h1', 'h2', 'strong', 'em', 'a', 'ul', 'li'],
    allowedAttrs: { 'a': ['href', 'title'] },
    stripEmpty: true,
    collapseWrappers: true
  }
});

API Reference

Complete reference for the Klar CMS JavaScript API.

Constructor#

Create a new Klar instance with configuration options:

javascript 
const editor = new Klar(options);

Options#

Option Type Default Description
target String | Element null CSS selector or DOM element to make editable
autoDiscover Boolean false Auto-find all [data-klar] elements
sanitize Boolean | Object true Enable HTML sanitization on save
mode String 'inline' Editor mode: 'inline' or 'block'
toolbar Boolean | Array true Show toolbar; or array of button names
onSave Function null Callback when content is saved
onChange Function null Callback on every content change

Instance Methods#

editor.enable()#

Enables editing mode on all registered regions.

editor.disable()#

Disables editing mode and cleans up editor state.

editor.save()#

Triggers save. Returns a Promise<Map<string, string>> of region IDs to clean HTML.

javascript 
const content = await editor.save();

content.forEach((html, regionId) => {
  console.log(`Region ${regionId}:`, html);
});

editor.getHTML(regionId?)#

Returns clean HTML for a specific region, or all regions if no ID is provided.

editor.destroy()#

Completely removes all Klar functionality and restores original DOM state. No artifacts left behind.

All methods that modify the DOM are synchronous. Only save() is async (when using a custom onSave handler).

Plugin System

Extend Klar's functionality with a simple, composable plugin API.

Overview#

Plugins let you extend Klar without modifying core code. Each plugin is a plain object with lifecycle hooks:

javascript 
const myPlugin = {
  name: 'my-plugin',

  // Called when Klar initializes
  init(editor) {
    console.log('Plugin loaded!');
  },

  // Transform HTML before save
  beforeSave(html) {
    return html.replace(/&nbsp;/g, ' ');
  },

  // Called after save completes
  afterSave(regions) {
    console.log('Saved', regions.size, 'regions');
  },

  // Cleanup when editor is destroyed
  destroy() {
    console.log('Plugin unloaded');
  }
};

new Klar({
  plugins: [myPlugin]
});

Available Hooks#

Hook Arguments Return Description
init editor — Called on initialization
beforeSave html html Transform HTML before sanitization
afterSave regions — Called after all regions are saved
onEnable — — Called when edit mode is enabled
onDisable — — Called when edit mode is disabled
destroy — — Cleanup on editor destroy

Editing Content

Learn how to create and manage editable regions in your HTML pages.

Editable Regions#

Any HTML element can become editable by adding the data-klar attribute with a unique identifier. Klar handles the rest — enabling contenteditable when editing, disabling it when done.

Inline vs Block Editing#

Inline mode edits text within existing elements. Block mode allows adding, removing, and reordering block-level elements like paragraphs, headings, and lists.

Events

Hook into editor lifecycle events for custom behavior.

Event List#

Klar emits events at key moments: enable, disable, change, save, error. Subscribe using the standard on() method.

javascript
editor.on('change', (regionId, html) => {
  console.log(`Changed: ${regionId}`);
});

editor.on('save', (regions) => {
  console.log('All regions saved');
});

Performance

How Klar stays fast and lightweight by design.

Bundle Size#

The full Klar runtime is 7.8KB gzipped. There are zero external dependencies. The sanitization engine uses the browser's built-in DOM parser rather than shipping a custom one.

Runtime Performance#

Klar batches DOM mutations using requestAnimationFrame and debounces change events. Sanitization runs in a document fragment to avoid layout thrashing.

Accessibility

Klar is built with WCAG 2.1 AA compliance in mind.

Keyboard Navigation#

All editor functionality is accessible via keyboard. The toolbar supports arrow key navigation and standard shortcuts like ⌘B for bold.

ARIA Attributes#

Editable regions announce their state via aria-label and role="textbox". The toolbar uses role="toolbar" with aria-pressed on toggle buttons.
